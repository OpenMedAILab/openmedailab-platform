# CHANGELOG

## 0.15.1 - 2026-06-19

### Changed

- 课题卡片标题前置展示课题编号，例如 `T0006: 医生复核证据压缩 RAG 用于抗 VEGF 长病程病例一页式摘要`。
- 组队人数超过目标时保留 `2/1` 计数和差异化颜色，不再追加额外文字提示。

## 0.15.0 - 2026-06-19

### Added

- 新增个人中心 `我的申请` 入口，集中展示参与、认领和资助申请的状态、审核意见、审核人 UID、审核时间和后续操作。
- 新增管理员申请审批弹窗，替代旧 `window.prompt` 审批方式，并支持预设审核意见。
- 新增申请审批浏览器验收截图，覆盖用户待审、管理员审批列表、审核弹窗和用户侧已通过回显。

### Changed

- 管理端协作审核入口从 `任务审批` 调整为 `申请审批`，避免与任务结果管理混淆。
- 资助意向在管理端按同一用户同一课题聚合展示，但仍逐条通过/拒绝和审计。
- 管理员申请审批列表改为待处理优先：pending 按创建时间正序，已处理按更新时间倒序。

### Fixed

- 管理员拒绝项目负责人认领、论文第一单位认领或资助意向时，审核意见必须非空；后端返回稳定 `review_note_required` 错误并保留双写字段错误结构。

## 0.14.0 - 2026-06-19

### Added

- 新增 Playwright 浏览器验收基线，覆盖课题详情协作 CTA、本人关系标签几何、移动首屏、资助 popover、不可认领提示、token 资助审批链路和基础无障碍行为。
- 新增统一 QA 脚本 `scripts/qa_full.sh`，串联后端检查、迁移检查、测试、前端构建、Playwright 和版本一致性校验。

### Changed

- 课题详情页补齐点赞、关注、参与、认领项目负责人、认领第一单位和资助等核心协作入口。
- 资助入口保持劳务费和算力两个主选项，token 资助通过“更多资助类型”渐进展开，并可在个人中心和管理端审批链路中验收。
- 本人关系标签在多视口下按斜角标签贴近对齐；超窄屏允许标签换行但不得重叠，标题继续与课题编号对齐。

### Fixed

- 修复资助 popover 在移动端为了完整入屏而远离触发按钮的问题，改为贴近按钮并在弹层内部滚动。
- 修复联系人悬浮卡、资助 popover、审批中按钮和本人相关课题 hover 顶线在真实浏览器验收中的响应式问题。
- 修复点赞排序在后端分页前生效，避免前端只对当前页局部重排。
- 修复并发参与、认领、资助和阶段变更在 SQLite/高并发锁冲突下可能返回 500 的问题，改为重试后返回稳定业务错误或幂等结果。
- 修复从课题详情页深链进入时，token 资助审批通过后无法在资助 popover 中恢复勾选并撤回的问题。
- 统一任务结果 PDF 与课题/主题 PDF 的 20MB 上传上限，并补齐超限上传失败审计。
- 统一管理端归档体验：归档不弹确认，物理删除继续弹确认。

## 0.13.1 - 2026-06-18

### Changed

- 项目负责人和论文第一单位认领在本人申请待审批时，课题卡按钮分别显示 `项目负责人审批中` 和 `第一单位认领审批中`，不再提前展示撤回认领入口。
- `claim_availability` 区分本人认领 `pending` 与 `approved`：待审批返回 `action=pending`，审核通过后才返回 `action=withdraw`。

### Fixed

- 修复待审批认领在课题卡上被误当作可撤回认领的问题，避免用户把“申请审批中”理解为“已正式认领”。
- 修复本人相关课题卡片顶部标签与斜角标签错位、长审批中文案在交互按钮中溢出的问题。
- 进一步收紧本人关系斜角标签与顶部小标签的水平距离，使顶部小标签贴近斜角标签视觉结束线。
- 修复课题卡底部操作按钮在桌面端被拉得过长、移动端过窄拥挤，以及资助浮层在窄视口或靠近底部时被遮挡的问题；首页主题入口在移动端改为横向卡片滚动。

## 0.13.0 - 2026-06-18

### Added

- 课题卡片新增登录用户本人关系提示：`我已认领`、`我已参与`、`我已资助`，并按该优先级置顶相关课题；本人相关课题 hover 顶线改为金色。
- 论文第一单位认领新增结构化“拟认领第一单位”字段和必填弹窗，审核列表、个人记录和审计 payload 均返回该字段。
- 认领按钮新增不可认领原因提示，覆盖未登录、阶段不可认领、席位已占用、积分不足和数据冲突等场景。
- 课题卡片资助入口改为贴近按钮的多选 popover，支持同时选择资助劳务费和资助算力，并保留资助 token 入口。

### Changed

- 项目负责人和论文第一单位认领改为同一课题同一席位最多一个 active 申请或通过记录；撤回或拒绝后释放席位。
- 所有面向用户的认领撤回文案改为“撤回项目负责人认领”或“撤回论文第一单位认领”。

## 0.12.0 - 2026-06-18

### Added

- 新增参与课题署名意向弹窗，支持 5 种论文署名意向并写入参与记录。
- 新增“认领课题（论文第一单位）”申请入口；项目负责人和论文第一单位认领进入管理员审批并返回审核意见。
- 新增首页登录态联系方式悬浮卡：创建者显示姓名，组队角色悬停显示成员姓名和微信。
- 新增自动积分规则：首次完善资料奖励 5 分、项目启动扣除参与人 50 分、进行中课题归档后返还参与人 100 分、用户可单次最多转赠 50 分。

### Changed

- 资助主入口收敛为资助算力、资助 token、资助劳务费三类；历史资助类型继续兼容旧数据。
- 管理端“任务审批”从仅审批资助意向调整为审批认领申请和资助意向。
- 参与和认领提交前会校验可用积分额度，已参与但未启动扣分的课题会预留 50 分额度。

## 0.7.0 - 2026-06-12

- 修复前端将自动通过的参与关系短暂显示为待处理的问题。
- 修复普通用户可通过接口把已进入管理员维护阶段的课题回退到开放招募的问题。
- 修复过长 `X-Request-ID` 可能写入审计日志失败的问题。
- 补齐管理员课题创建/更新失败路径的审计日志。
- 补齐注册、登录、退出、强制改密和资料修改等用户生命周期动作的审计日志。
- 修复自动生成课题 ID 遇到唯一约束冲突时缺少重试保护的问题。
